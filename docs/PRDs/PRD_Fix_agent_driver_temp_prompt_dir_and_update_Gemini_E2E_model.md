# PRD: Fix agent_driver temp prompt dir and update Gemini E2E model

## 1. Context & Problem (业务背景与核心痛点)
When running SDLC with `LLM_DRIVER=gemini`, the Gemini CLI refuses to read the temporary prompt file generated by `agent_driver.py`. This happens because `agent_driver.py` hardcodes the temp directory to `/tmp` (`tempfile.mkstemp(..., dir='/tmp')`), which violates the Gemini CLI's strict workspace sandboxing policies (restricted to `/root/.openclaw/workspace`).
Additionally, the standalone E2E test (`scripts/e2e/e2e_test_agent_driver_gemini.sh`) uses `google/gemini-2.0-flash` as the fallback model, which currently returns a 404 `ModelNotFoundError` from the Google API.

## 2. Requirements & User Stories (需求定义)
- **Fix Prompt Directory**: Modify `scripts/agent_driver.py` so that the temporary prompt text file is written to a `.tmp` directory within the project's workspace (e.g. at the project root) instead of the system `/tmp`. Ensure the file is properly cleaned up after invocation.
- **Update Test Model**: Update the fallback `TEST_MODEL` in `scripts/e2e/e2e_test_agent_driver_gemini.sh` from `google/gemini-2.0-flash` to `gemini-2.5-flash`.
- **E2E Success**: The `e2e_test_agent_driver_gemini.sh` test must pass cleanly without directory traversal or 404 errors.

## 3. Architecture & Technical Strategy (架构设计与技术路线)
- **Target File 1: `scripts/agent_driver.py`**
  - **Strategy**: Inside `invoke_agent`, locate the project root using `os.path.dirname(RUNTIME_DIR)`. Construct a `temp_dir` path: `os.path.join(os.path.dirname(RUNTIME_DIR), ".tmp")`. Create this directory if it doesn't exist (`os.makedirs`).
  - Pass `dir=temp_dir` to `tempfile.mkstemp` instead of `dir="/tmp"`.
  - The cleanup logic `os.remove(path)` inside the `finally` block remains unchanged and handles removal automatically.
- **Target File 2: `scripts/e2e/e2e_test_agent_driver_gemini.sh`**
  - **Strategy**: Update the `TEST_MODEL` default value to `gemini-2.5-flash`.

## 4. Acceptance Criteria (BDD 黑盒验收标准)
- **Scenario 1:** Gemini E2E test execution with default parameters.
  - **Given** the modified `agent_driver.py` and updated `e2e_test_agent_driver_gemini.sh` script
  - **When** the developer executes `bash scripts/e2e/e2e_test_agent_driver_gemini.sh`
  - **Then** the test should successfully invoke the `gemini-2.5-flash` model, verify the session creation, and exit with status code 0 (`✅ Isolated E2E Test Passed`), without any 404 or workspace permission errors.

## 5. Overall Test Strategy & Quality Goal (测试策略与质量目标)
- E2E testing using the existing script `e2e_test_agent_driver_gemini.sh`.
- The main goal is to ensure Gemini CLI integration works correctly without sandboxing violations, and the test itself uses a valid model.

## 6. Framework Modifications (框架防篡改声明)
- `scripts/agent_driver.py`
- `scripts/e2e/e2e_test_agent_driver_gemini.sh`

## 7. Hardcoded Content (硬编码内容)
- **For `scripts/e2e/e2e_test_agent_driver_gemini.sh` fallback model assignment (Exact Text):**
```bash
export TEST_MODEL="${1:-${TEST_MODEL:-gemini-2.5-flash}}"
```
