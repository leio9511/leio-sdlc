# PR-001a-2: Implement Error Handling in safe_git_checkout

## 1. Objective
Enhance `safe_git_checkout` to catch `subprocess.CalledProcessError` and raise the custom `GitCheckoutError` without executing destructive commands.

## 2. Scope & Implementation Details
- Modify `safe_git_checkout(branch_name)` in the git utility module.
- Wrap the `subprocess` call in a `try...except subprocess.CalledProcessError` block.
- In the `except` block, log the error clearly and raise `GitCheckoutError`.

## 3. TDD & Acceptance Criteria
- In the test file, write a test `test_safe_git_checkout_failure`.
- Call `safe_git_checkout` with a non-existent branch (or mock `subprocess.run` to raise `CalledProcessError`).
- Assert that `GitCheckoutError` is raised.
