# PRD: Enforce SDLC via Git Pre-Commit Hook (ISSUE-1012)

## Context
As learned from a recent failure incident, the soft constraints (prompt-level rules like `SOUL.md`) applied to the main OpenClaw Agent (Manager) are insufficient to prevent it from directly executing `write` or `exec` commands to manipulate source code. To enforce the "Lobster Architecture" and strictly decouple roles, we need a hard physical isolation mechanism.

## Requirements
- Introduce a standardized Git pre-commit hook into the `leio-sdlc` project (and ultimately globally).
- The hook must strictly intercept and reject any commits made directly to protected branches (e.g., `master`, `main`) unless they are generated by the official SDLC Orchestrator process.
- The hook should evaluate an environment variable (e.g., `SDLC_ORCHESTRATOR_RUNNING=1`) or verify the presence of an active SDLC orchestration state before allowing a commit.
- If the Manager (main Agent) attempts to commit code manually via CLI, the pre-commit hook MUST reject it with a clear echo message: `"ERROR: SDLC Violation! Direct commits are forbidden. You must use orchestrator.py."`
- Provide a `patch_guardrails.sh` or an installation script to automatically bind this hook into `.git/hooks/pre-commit` upon deployment.

## Framework Modifications
- `scripts/orchestrator.py` (Must inject the `SDLC_ORCHESTRATOR_RUNNING` environment variable during subprocess calls).
- `scripts/merge_code.py` (Ensure the environment is passed when finalizing PRs).
- `TEMPLATES/AgentSkill_Archetype/deploy.sh` (To register the hook during installation).
- `scripts/install_hook.sh` (New file for the git hook payload).

## Architecture
This relies on native Git hooks for enforcement. By making the Orchestrator inject a secure environment variable or token, the hook mathematically prevents the Main Agent from bypassing the SDLC flow, resolving the "Tool Affordance" issue at the lowest permission level.

## Acceptance Criteria
- [ ] A pre-commit hook is automatically installed into the repository.
- [ ] A manual `git commit` run by the Main Agent is rejected with the specified error message.
- [ ] Commits executed by the Coder, Reviewer, or Orchestrator successfully pass the hook.
