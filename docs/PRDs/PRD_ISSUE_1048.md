# PRD: ISSUE-1048 (Local Sandbox & Micro-PR fixing)

## 1. Context & Problem Definition (核心问题与前因后果)
The `leio-sdlc` system currently requires a strict minimum of 2 PRs in `spawn_planner.py`. This is overly restrictive for simple tasks, leading to unnecessary fragmentation and token bloat. Additionally, the system lacks a proper local sandbox, causing state and job files to clutter the repository root or `.git` directory unintentionally. We need a "Local Invisible Sandbox" under `.sdlc_runs/` to isolate SDLC executions without affecting the developer's normal git workflow, while keeping compatibility with `v0.8.1` (pre-polyrepo architecture). 

The initial proposal of a completely flat `.sdlc_runs/` broke the orchestrator's `job_dir` logic (which relies on being able to safely read `*.md` and perform `shutil.rmtree` during replanning without destroying other PRDs' states).

## 2. Requirements (需求说明)
1. **Micro-PR Restraint**: Relax the 2-PR minimum in `spawn_planner.py` to allow 1 PR for simple tasks.
2. **Retain Strict Minimum for Slicing**: The strict minimum must be retained for the `--slice-failed-pr` flag.
3. **Local Invisible Sandbox Initialization**: Modify `orchestrator.py` to explicitly append `.sdlc_runs/` to `.git/info/exclude` at startup.
4. **State/Job Routing**: Modify `orchestrator.py`, `spawn_planner.py`, and `create_pr_contract.py` to route all jobs and states locally to `.sdlc_runs/`.
5. **PRD-Specific Sandbox Structure (CRITICAL)**: The sandbox structure must be: `.sdlc_runs/<prd_name>/` (this becomes the `job_dir`). 
6. **Flat PR Contracts**: The Planner MUST still write its PR contracts directly into this `job_dir` (flat within that specific PRD's folder). The Planner must NOT append any further subdirectories inside the `job_dir` (e.g., do NOT append the project name like `.sdlc_runs/<prd_name>/<project_name>/`). 
7. **Implementation Freedom**: The Coder has full implementation freedom regarding the internal mechanisms of these changes as long as the architectural boundaries are respected. Specifically, the Coder should modify `create_pr_contract.py` to ignore the `--project` parameter for path construction, OR ensure the Planner is instructed NOT to use the `--project` parameter when generating contracts.

## 3. Architecture (架构设计)
We are implementing a **Local Invisible Sandbox Pattern** to ensure zero pollution of the user's working tree while maintaining observability for the AI agents.
- **Git Exclude**: `.git/info/exclude` is preferred over `.gitignore` because `.sdlc_runs` is a local framework-specific directory and shouldn't be committed or enforced on other contributors' environments.
- **PRD-Isolated Job Directory**: The system will use `.sdlc_runs/<prd_name>/` as the `job_dir`. This allows the orchestrator to safely isolate states per PRD, and safely wipe (`shutil.rmtree`) the directory during a replan without affecting concurrent jobs.
- **Flat PR Contracts**: Inside `.sdlc_runs/<prd_name>/`, all generated PR contract markdown files must sit flat. No further nested subdirectories (like `<project_name>`) are allowed, ensuring the v0.8.1 orchestrator's shallow glob can easily find them.

## 4. Acceptance Criteria (验收标准)
- [ ] `spawn_planner.py` allows generating a single PR for simple tasks.
- [ ] `spawn_planner.py` requires >= 2 PRs when invoked with `--slice-failed-pr`.
- [ ] Running `orchestrator.py` automatically appends `.sdlc_runs/` to `.git/info/exclude` if it's not already there.
- [ ] The orchestrator creates a PRD-specific directory at `.sdlc_runs/<prd_name>/` for the current run.
- [ ] AI artifacts (jobs, states, contracts) generated by the orchestrator and planner are saved inside `.sdlc_runs/<prd_name>/`.
- [ ] PR contracts are stored flat inside `.sdlc_runs/<prd_name>/` without further subdirectories (like `/leio-sdlc/`).

## 5. Framework Modifications (框架修改声明)

- `scripts/create_pr_contract.py`
