# PR-1: Refactor spawn_coder to inject coder_playbook.md

## 1. Objective
Refactor `spawn_coder.py` to physically read the `playbooks/coder_playbook.md` file and inject its content directly into the `task_string` prompt, explicitly forbidding the modification of the markdown file's `status` field.

## 2. Scope & Implementation Details
- In `scripts/spawn_coder.py`:
  - Locate `playbooks/coder_playbook.md` relative to the script file using `__file__`.
  - Read the content of the playbook.
  - Inject the content into the `task_string` variable.
  - Add explicit instruction in the prompt: "You are strictly forbidden from manually editing the markdown file's `status` field."

## 3. TDD & Acceptance Criteria
- Create/update a test (e.g., in `tests/test_spawn_coder.py`) that mocks the playbook file read and verifies that `task_string` contains the playbook content and the forbidden status modification instruction.
- The test must pass during `./preflight.sh`.
