# PRD: Deploy_and_Auditor_Fixes

## 1. Context & Problem (业务背景与核心痛点)
1. **Deployment Pollution**: The current `deploy.sh` blindly copies temporary artifacts (e.g., `.sdlc_runs/`, logs) to production, causing disk bloat.
2. **Silent Auditor Failure**: If a PRD fails structural validation in `spawn_auditor.py`, the script exits without calling `notify_channel`, causing the user to wait indefinitely.
3. **Template Disconnect**: The `PRD.md.template` is missing Section 7, but `spawn_auditor.py` mandatory requires it, leading to inevitable rejections for compliant agents.
4. **Unclear Auditor Guidance**: Current error messages don't explicitly warn agents against overwriting templates with raw `write` tools.

## 2. Requirements & User Stories (需求定义)
- **R1**: Refactor `deploy.sh` to use double `exclude-from` flags (`.gitignore` and `.release_ignore`).
- **R2**: Update `.release_ignore` to filter out all development/test assets.
- **R3**: Patch `spawn_auditor.py` to ensure `notify_channel` is called on structural validation failure.
- **R4**: Enhance Auditor error messages to provide explicit guidance on template preservation.
- **R5**: Update the official `PRD.md.template` to include the missing Section 7 with usage instructions and examples.

## 3. Architecture & Technical Strategy (架构设计与技术路线)
- **Double-Funnel Deployment**: Replace the existing `rsync` line with:
  `rsync -a --exclude-from='.gitignore' --exclude-from='.release_ignore' . "$TMP_DIR/"`
- **Notification Patch**: Wrap structural checks in `spawn_auditor.py` with a helper that triggers `agent_driver.notify_channel` before `sys.exit(0)`.
- **Template Synchronization**: Append the "Hardcoded Content" section to the end of the global and local PRD templates to align with Auditor logic.

## 4. Acceptance Criteria (BDD 黑盒验收标准)
- **Scenario 1: Clean Deployment**
  - **Given** a dirty workspace with `.sdlc_runs/` and `tests/`
  - **When** running `deploy.sh`
  - **Then** the production directory must contain neither.
- **Scenario 2: Audible Auditor Failure**
  - **Given** a malformed PRD
  - **When** running `spawn_auditor.py`
  - **Then** the user MUST receive a WeChat notification explaining the structural error and corrective steps.

## 5. Overall Test Strategy & Quality Goal (测试策略与质量目标)
- **Unit Testing**: Assert that `spawn_auditor.py` correctly triggers notifications for missing PRD sections.
- **Functional Testing**: Simulate a deployment and verify the exclusion list correctly handles both untracked (gitignore) and tracked (release_ignore) files.
