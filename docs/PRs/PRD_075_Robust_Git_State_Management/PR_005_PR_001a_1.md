# PR-001a-1: Define GitCheckoutError and Base Function

## 1. Objective
Define the custom exception `GitCheckoutError` and create the base `safe_git_checkout(branch_name)` function to execute a successful git checkout.

## 2. Scope & Implementation Details
- In the git utility module (e.g., `git_utils.py`), define a new exception class `GitCheckoutError`.
- Implement `safe_git_checkout(branch_name)` which uses `subprocess.run` to execute `git checkout <branch_name>`.

## 3. TDD & Acceptance Criteria
- Create a test file (e.g., `test_git_utils.py`) if it doesn't exist.
- Write a test `test_safe_git_checkout_success` that calls `safe_git_checkout` with a valid branch (or mocks `subprocess.run` to succeed).
- Assert that the function completes without raising an exception.
