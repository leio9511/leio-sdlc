# PR-2: Refactor spawn_reviewer and spawn_planner to inject playbooks

## 1. Objective
Refactor `spawn_reviewer.py` and `spawn_planner.py` to physically read their respective playbooks (`reviewer_playbook.md` and `planner_playbook.md`) and inject the content into the `task_string` prompt, along with the forbidden status edit rule.

## 2. Scope & Implementation Details
- In `scripts/spawn_reviewer.py` and `scripts/spawn_planner.py`:
  - Locate the respective playbook in `playbooks/` relative to `__file__`.
  - Read the content of the playbook.
  - Inject the content into the `task_string` variable.
  - Add explicit instruction in the prompt forbidding manual edits to the `status` field.

## 3. TDD & Acceptance Criteria
- Create/update tests (e.g., in `tests/test_spawn_reviewer.py` and `tests/test_spawn_planner.py`) to mock the playbook file read and verify that `task_string` contains the corresponding playbook content and the forbidden status modification instruction.
- The tests must pass during `./preflight.sh`.
