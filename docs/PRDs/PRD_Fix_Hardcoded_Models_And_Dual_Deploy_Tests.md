# PRD: Fix_Hardcoded_Models_And_Dual_Deploy_Tests

## 1. Context & Problem (业务背景与核心痛点)
1. In recent changes (PR-001, PR-002), the dual compatibility deployment logic was added to the main `deploy.sh` and `pm-skill/deploy.sh`. However, the comprehensive integration tests ensuring `deploy.sh` correctly copies `.dist/scripts/` to `.openclaw/skills/` without dropping `.py` or `.sh` files were missing or incomplete.
2. The `SDLC_MODEL` defaults in `scripts/agent_driver.py` and test mocks in `tests/test_gemini_agent_driver.py` were improperly hardcoded in the driver logic itself. This violates the Configuration Externalization anti-pattern. 
3. **Workspace Pollution**: The `.tmp` directory for JIT prompt injection is incorrectly placed inside the `project_root` (e.g., `leio-sdlc/.tmp`), which pollutes the local workspace. It should be placed securely under the `global_dir` (e.g., `~/.openclaw/workspace/.tmp`).
4. **Tool Use Hallucination (Gemini CLI)**: The `gemini` CLI driver branch injects massive prompts via the `-p` direct argument. This causes headless `gemini` to treat the prompt as a text-completion task rather than an Agentic Tool Loop, skipping file generation. It needs to utilize the same "File Indirection" (`secure_msg`) approach as `openclaw agent`.
5. **Session Management (Gemini CLI)**: The `gemini` CLI does not support initializing a session via an externally provided `--session-id`. Thus, stateful agents (like Coder) cannot resume properly unless the internal session UUID generated by `gemini` is captured and managed.

## 2. Requirements & User Stories (需求定义)
1. **Extract Magic Strings to Config Layer**: Create a new centralized configuration file `scripts/config.py` to hold system constants like `DEFAULT_GEMINI_MODEL`.
2. **Global Temp Directory Fix**: Refactor `scripts/agent_driver.py` so that the `temp_dir` for `sdlc_prompt_*.txt` files resolves to `~/.openclaw/workspace/.tmp` (global directory) instead of polluting `project_root`.
3. **File Indirection for Gemini CLI**: Modify `agent_driver.py` so that `gemini` CLI is invoked with `-p secure_msg` (pointing to the temp file) instead of `-p task_string`.
4. **Session Resumption for Gemini Coder**: Implement logic in `agent_driver.py` to detect `session_key`. If the driver is `gemini` and a session key exists, use `-r <session_key>` instead of `-p`. Note: This requires upstream callers (like Coder) to capture the initial Gemini UUID if we want true continuity, but for now, we ensure the driver structure supports `-r`. 
5. **Fix Missing Dual Deploy Tests**: Implement bash-based test cases (e.g., `tests/test_034_dual_deploy.sh` or update existing tests) to verify the dual deployment logic in both the main `leio-sdlc/deploy.sh` and `skills/pm-skill/deploy.sh`.

## 3. Architecture & Technical Strategy (架构设计与技术路线)
- **Configuration Externalization (配置隔离)**: 
  - Create `scripts/config.py` and define `DEFAULT_GEMINI_MODEL = "gemini-3.1-pro-preview"`.
  - In `scripts/agent_driver.py`, import this module and use `config.DEFAULT_GEMINI_MODEL`.
- **Global Temp Directory**: Change `temp_dir = os.path.join(project_root, ".tmp")` to `temp_dir = os.path.expanduser("~/.openclaw/workspace/.tmp")` in `scripts/agent_driver.py`.
- **Agent Driver Prompt Unification**:
  - `cmd = [cmd_exec, "--yolo", "-p", secure_msg, "--model", model]` (for new sessions).
  - If a specific resumption is requested (e.g., Coder iterative loops), inject `-r session_key`.
- **Sandbox Testing for Bash**: Create/update bash test scripts to dry-run the deployment scripts (`deploy.sh`) using isolated temporary directories (Sandbox Pattern).

## 4. Acceptance Criteria (BDD 黑盒验收标准)

- **Scenario 1:** Temp prompt files are created in the global workspace.
  - **Given** an SDLC run.
  - **When** `agent_driver.py` generates the `sdlc_prompt_xxx.txt`.
  - **Then** the file is located in `~/.openclaw/workspace/.tmp` and the project repository remains clean.

- **Scenario 2:** Gemini CLI behaves agentically via File Indirection.
  - **Given** `LLM_DRIVER=gemini`.
  - **When** `agent_driver.py` invokes the CLI.
  - **Then** the CLI command uses `-p "Read your complete task instructions from ..."` rather than the raw payload.

- **Scenario 3:** Dual deployment is fully tested in an isolated sandbox.
  - **Given** the `leio-sdlc` deployment scripts.
  - **When** `test_034_dual_deploy.sh` is executed.
  - **Then** it passes successfully without polluting system paths.

## 5. Overall Test Strategy & Quality Goal (测试策略与质量目标)
- **Unit Testing**: Python unit tests in `tests/test_gemini_agent_driver.py` must mock the environment to test the config fallback, global temp path logic, and `-p` secure_msg formatting.
- **Sandbox Integration Testing**: Bash scripts must simulate the deployment environment using isolated temporary directories (Sandbox Pattern).
